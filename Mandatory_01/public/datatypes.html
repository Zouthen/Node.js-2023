Data types

- String => This is for handling text or words.

- Number => This is for handling numbers.

- Boolean => This is for handling true or false.

- BigInteger => This is for handling big numbers.

- undefined => This is for handling undefined values.

- null => This is for handling null values.

- Object => This is for handling objects.

- Symbol => This is for handling symbols.


Objects: object, Array, Date

- Object => An Object is like a container that holds data in key-value pairs. Keys are strings, and values can be of any data type, including other objects.

- Array => An Array is a special type of object that holds multiple values in a single variable.

- Date => The Date object is used to work with dates and times.


Type coercion

Implicit Type Coercion: This happens automatically when you perform operations between values of different types. JavaScript tries to convert one or both of the values to a common type before performing the operation.

Explicit Type Coercion: This happens when you use a method like Number() or String() to explicitly convert a value to a specific type.


typeof

The typeof operator in JavaScript is used to determine the data type of a given variable or expression. It returns a string indicating the type of the operand.


== vs ===

Loose Equality (==): The == operator performs type coercion if the operands are of different types. For example, "5" == 5 will return true because the string is coerced to a number before comparison.

Strict Equality (===): he === operator, on the other hand, does not perform type coercion. It checks both the value and the type of the operands. If they are of different types, the comparison returns false without attempting to convert them.


truthy vs falsy

Truthy Values: Values that evaluate to true when used in a boolean context. Examples of truthy values include: - Non-empty strings: "hello", "123". - Non-zero numbers: 42, -3.14. - Arrays and objects: [], {}. - The special value true.

Falsy Values: Values that evaluate to false when used in a boolean context. Examples of falsy values include: - The empty string: "". - The number 0 and NaN (Not-a-Number). - null and undefined. - The special value false.


NaN

NaN stands for "Not-a-Number" and is a special value in JavaScript that represents the result of an operation that should return a number but doesn't.
